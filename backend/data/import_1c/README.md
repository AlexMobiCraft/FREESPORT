# Директория для импорта данных из 1С

Эта директория используется для хранения XML файлов, выгруженных из системы 1С, перед импортом в базу данных FREESPORT.

## Требуемая структура директорий

Перед запуском импорта через админ-панель убедитесь, что данные из 1С выгружены в следующей структуре:

```
data/import_1c/
├── contragents/          # XML файлы с контрагентами (клиентами)
│   └── contragents_*.xml
├── goods/                # XML файлы с товарами
│   └── goods_*.xml
├── offers/               # XML файлы с предложениями
│   ├── offers_*.xml
│   └── import_files/     # Изображения товаров
├── prices/               # XML файлы с ценами
│   └── prices_*.xml
├── rests/                # XML файлы с остатками
│   └── rests_*.xml
├── priceLists/           # Прайс-листы
├── storages/             # Склады
└── units/                # Единицы измерения
```

## Типы импорта

### 1. Полный каталог
**Требуемые директории:** `goods/`, `offers/`, `prices/`, `rests/`, `priceLists/`

Импортирует все данные:
- Товары и их характеристики
- Категории и бренды
- Цены для всех уровней пользователей
- Остатки на складах
- Справочники (единицы измерения, склады)

### 2. Только остатки
**Требуемые директории:** `rests/`

Обновляет только остатки товаров на складах.
⚠️ Требует предварительно импортированный каталог товаров.

### 3. Только цены
**Требуемые директории:** `prices/`

Обновляет только цены товаров для всех уровней пользователей.
⚠️ Требует предварительно импортированный каталог товаров.

### 4. Клиенты
**Требуемые директории:** `contragents/`

Импортирует контрагентов (клиентов) из 1С:
- Организации (ООО, ИП)
- Физические лица
- Реквизиты и контактные данные

## Как запустить импорт

1. **Выгрузите данные из 1С** в эту директорию с соблюдением структуры выше

2. **Откройте админ-панель** Django:
   ```
   http://localhost:8001/admin/
   ```

3. **Перейдите в раздел "ИНТЕГРАЦИИ"** → **"Импорт из 1С"**

4. **Выберите тип импорта** и нажмите "Запустить импорт"

5. **Отслеживайте прогресс** на странице "Сессии импорта"

## Проверка данных

После выгрузки данных из 1С проверьте:

```bash
# Проверить наличие директорий
ls -la data/import_1c/

# Проверить наличие XML файлов
find data/import_1c/ -name "*.xml" | head -10

# Проверить размер файлов
du -sh data/import_1c/*
```

## Устранение проблем

### Ошибка: "Отсутствуют обязательные поддиректории"

**Причина:** Данные не выгружены из 1С или выгружены в неправильной структуре.

**Решение:**
1. Убедитесь, что выгрузка из 1С завершилась успешно
2. Проверьте наличие всех требуемых поддиректорий
3. Проверьте наличие XML файлов в директориях

### Ошибка: "Невозможно загрузить остатки/цены: каталог товаров пуст"

**Причина:** Пытаетесь импортировать остатки или цены, но в базе данных нет товаров.

**Решение:**
1. Сначала выполните "Полный каталог"
2. Затем импортируйте остатки или цены

### Ошибка: "Импорт уже запущен"

**Причина:** В данный момент уже выполняется другой импорт.

**Решение:**
1. Дождитесь завершения текущего импорта
2. Проверьте статус на странице "Сессии импорта"
3. Если импорт завис (более 1 часа), обратитесь к администратору

## Дополнительная информация

- **Формат данных:** CommerceML 3.1 (XML)
- **Кодировка:** UTF-8
- **Максимальный размер файла:** без ограничений
- **Timeout импорта:** 1 час (Redis lock)

## Поддержка

При возникновении проблем проверьте логи:
- Django: `backend/logs/django.log`
- Celery: `backend/logs/celery.log`

Или обратитесь к администратору системы.
