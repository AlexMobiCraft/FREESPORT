# Technical Specification: 1C Import Bugs Fixes

## 1. Введение (Introduction)
Данная спецификация описывает решения для двух багов, возникающих после выполнения процесса импорта товаров и категорий из 1С в систему FREESPORT.
1. Исчезновение избранных брендов с главной страницы.
2. Сохранение и отображение неактуальных (лишних) категорий в каталоге.

## 2. Анализ проблемы
### 2.1. Исчезающие бренды
На главной странице отображаются бренды, у которых установлен флаг `is_featured=True`.
После импорта из 1С этот список может повреждаться или сбрасываться по ряду причин:
*   При добавлении поля `normalized_name` старые бренды не получили это значение. В `variant_import.py` дедупликация ищет бренды только по `normalized_name`. Если оно пустое, то старый (избранный) бренд не находится, и создается новый дубликат (по умолчанию `is_featured=False`).
*   Список `featured` брендов кэшируется на фронте/бэке через `FEATURED_BRANDS_CACHE_KEY`. Инвалидация этого кэша настроена через Django Signals. Метод `update_fields` или операции вроде `bulk_create`/`bulk_update` обходят сигналы, поэтому кэш не инвалидируется после массового импорта 1С.

### 2.2. Лишние уровни абстракции категорий
Ранее был внедрен скрипт, отсекающий из 1С все категории, которые не принадлежат корневой якорной категории (например, "СПОРТ").
Однако, новые ограничения работают только на момент *создания/обновления*. Если категория уже была в базе и была удалена в 1С (или перестала подходить под текущие фильтры), ее статус `is_active=True` в базе сохраняется, и она продолжает отображаться на сайте.

## 3. Предлагаемое решение

### 3.1. Улучшение импорта брендов
*   **Файл**: `backend/apps/products/services/variant_import.py`
*   **Метод**: `process_brands`
*   **Изменение 1**: Добавить fallback-поиск бренда: если бренд не найден по `normalized_name`, попробовать найти его по `name__iexact=onec_name`.
*   **Изменение 2**: После успешного завершения обработки всех брендов (или в конце операции импорта) добавить строчку для принудительной инвалидации кэша избранных брендов.
    ```python
    from django.core.cache import cache
    from apps.products.constants import FEATURED_BRANDS_CACHE_KEY

    # ... логика импорта ...
    cache.delete(FEATURED_BRANDS_CACHE_KEY)
    ```

### 3.2. Деактивация лишних категорий
*   **Файл**: `backend/apps/products/services/variant_import.py`
*   **Метод**: `process_categories`
*   **Изменение**: Собрать все `onec_id` из текущего XML файла, которые успешно прошли фильтрацию.
    Затем выполнить деактивацию устаревших категорий (у которых есть `onec_id` в БД, но их нет в текущем валидном списке из XML).
    ```python
    # Предполагаемый код после завершения создания и обновления категорий (Шаг 1 и 2):
    valid_onec_ids = list(category_map.keys())
    if valid_onec_ids:
        obsolete_categories_updated = Category.objects.filter(
            onec_id__isnull=False
        ).exclude(
            onec_id__in=valid_onec_ids
        ).update(is_active=False)
        logger.info(f"Deactivated {obsolete_categories_updated} obsolete categories.")
    ```

## 4. План тестирования (QA)
*   **Unit Tests**: Запустить бэкенд тесты (`pytest`) для проверки корректности существующих функций.
*   **Интеграционный тест (бренды)**: 
    * Установить флаг `is_featured=True` для старого бренда без `normalized_name`.
    * Запустить импорт из 1С. Убедиться, что дубликат не создан, а флаг остался `True`.
    * Убедиться, что кэш API `brands/featured/` обновился.
*   **Интеграционный тест (категории)**:
    * Создать тестовую категорию с `onec_id=test_id` и `is_active=True`.
    * Запустить импорт 1С. В XML файле не должно быть `test_id`.
    * Убедиться, что после импорта тестовая категория стала `is_active=False`.

* Оценка времени: 30-40 минут на реализацию и тестирование.
