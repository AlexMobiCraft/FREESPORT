# План разработки Frontend Эпиков 10-20

**Дата создания:** 2025-11-15  
**Статус:** В разработке  
**Фазы:** Фаза 1 (Фундамент) + Фаза 2 (Реализация страниц и пользовательских сценариев)  
**Общая продолжительность:** 28 недель (4 недели Фаза 1 + 24 недели Фаза 2)  
**Команда:** 1 фулстек разработчик

---

## Контекст

Данный план описывает разработку 11 frontend-эпиков (10-20) для платформы FREESPORT, которые реализуют фундамент, все ключевые страницы и пользовательские сценарии согласно frontend-development-plan.md и front-end-spec.md.

**Предварительные условия:**

- ✅ Эпики 1-8 завершены (фундамент, API, интеграция с 1С, базовый UI)
- ✅ Эпик 9 завершён (административная панель)
- ✅ Backend API готов и задокументирован (api-spec.yaml)
- ✅ Дизайн-система определена (design-system.json)

**Технологический стек:**

- Next.js 14+ App Router
- TypeScript 5.0+
- Zustand (state management)
- Axios (API client)
- Jest + React Testing Library (unit/integration тесты)
- Playwright (E2E тесты)
- Tailwind CSS + design-system tokens

### **Фаза 1: Фундамент (4 недели)**

#### **Эпик 10: Фундамент и подготовка**

- **Приоритет:** P0 (критичный, блокирующий)
- **Длительность:** 4 недели
- **Зависимости:** Эпики 1-2 (Backend API готов)

**Документы для разработки:**

- `docs/frontend-development-plan.md` (раздел 2, строки 13-37)
- `docs/front-end-spec.md` (раздел "Общие цели и принципы UX", строки 5-42)
- `frontend/docs/design-system.json` (весь документ — основа UI Kit)
- `frontend/docs/testing-standards.md` (раздел "Конфигурация Jest", строки 161-199)
- `frontend/docs/testing-typescript-recommendations.md` (весь документ)
- `docs/api-spec.yaml` (для генерации TypeScript типов)
- `frontend/docs/design-system.json` (для реализации базового UI-Kit)

**Scope:**

**2.1. Подготовка окружения и качества кода:**

- Проверка работоспособности Docker-окружения (`make up`)
- Синхронизация переменных в `frontend/.env.local`
- Настройка ESLint (`eslint.config.mjs`) с плагинами `react-hooks` и `jsx-a11y`
- Настройка Prettier и `lint-staged` для pre-commit hooks
- Настройка CI/CD для автоматического запуска линтеров

**2.2. Архитектура и UI Kit:**

- Создание глобального `layout.tsx` с провайдерами (тема, состояние)
- Реализация базового UI-Kit в `src/components/ui/` согласно `frontend/docs/design-system.json`:
  - Button (все варианты: primary, secondary, tertiary, subtle)
  - Input, Select, SearchField
  - Checkbox, Toggle
  - Modal, Card, Badge, Tag, Chip
  - Breadcrumb, Tabs
  - Spinner, InfoPanel, SupportPanel
  - Все компоненты должны использовать токены из design-system.json (colors, typography, spacing, radius, shadows)
- Настройка Zustand stores:
  - `authStore` (user, login, logout, refresh token)
  - `cartStore` (items, addItem, removeItem, updateQuantity)
- Создание shared utilities и helpers

**2.3. Слой взаимодействия с API:**

- Централизованный axios клиент `src/services/api-client.ts`:
  - Интерцепторы для автоматического добавления JWT
  - Обработка ошибки 401 (обновление токена)
  - Обработка сетевых ошибок
  - Retry логика
- Генерация TypeScript типов из `api-spec.yaml`
- Реализация сервисных функций для эндпоинтов:
  - `authService` (login, register, logout, refresh)
  - `productsService` (getAll, getById, search, filter)
  - `categoriesService` (getAll, getTree)
  - `cartService` (get, add, update, remove, applyPromo)
  - `ordersService` (create, getAll, getById)
- Unit-тесты для всех сервисов с MSW моками

**Критерии завершения:**

- ✅ Docker окружение работает стабильно
- ✅ ESLint/Prettier настроены и проходят проверки
- ✅ Все базовые UI компоненты реализованы и протестированы
- ✅ Zustand stores работают корректно
- ✅ API клиент с интерцепторами функционирует
- ✅ TypeScript типы сгенерированы для всех API
- ✅ Unit-тесты покрытие 80%+
- ✅ CI/CD pipeline настроен и работает
