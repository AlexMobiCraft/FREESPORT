## Story

**As a** администратор системы,
**I want** выбрать тип импорта "Только изображения товаров" в админ-панели Django,
**so that** я могу обновить изображения товаров из директории 1С без полного импорта каталога.

5. Валидация зависимостей проверяет наличие каталога товаров
6. UI следует существующим паттернам админ-панели

#### 3. Валидация зависимостей

**Местоположение**: [views.py:144-161](../../../backend/apps/integrations/views.py#L144-L161)

**Существующий паттерн**:

```python
def _validate_dependencies(selected_types: list[str]) -> tuple[bool, str]:
    """
    Проверка зависимостей между типами импорта.

    Returns:
        Tuple[bool, str]: (is_valid, error_message)
    """
    if "stocks" in selected_types or "prices" in selected_types:
        if not Product.objects.exists():
            return (
                False,
                "⚠️ Невозможно загрузить остатки/цены: "
                "каталог товаров пуст. Сначала импортируйте полный каталог.",
            )
    return True, ""
```

**Расширить для "images"**:

```python
def _validate_dependencies(selected_types: list[str]) -> tuple[bool, str]:
    # Типы, требующие наличия товаров в БД
    catalog_dependent_types = ["stocks", "prices", "images"]

    if any(t in selected_types for t in catalog_dependent_types):
        if not Product.objects.exists():
            return (
                False,
                "⚠️ Невозможно загрузить остатки/цены/изображения: "
                "каталог товаров пуст. Сначала импортируйте полный каталог.",
            )
    return True, ""
```

### Важные замечания

1. **Story 15.1 НЕ включает Celery задачу**:
   - Story 15.1 только добавляет UI и валидацию
   - Фактическая обработка изображений будет в Story 15.2
   - После Story 15.1 админ сможет выбрать тип "images", но задача завершится с ошибкой "Unknown import type"

2. **Совместимость с существующими типами**:
   - Новый тип не должен влиять на работу catalog, stocks, prices, customers
   - Regression тесты должны проверить, что все старые типы работают как прежде

3. **Миграция БД**:
   - PostgreSQL поддерживает добавление новых значений в enum через миграцию
   - Используйте `migrations.RunSQL` для безопасного добавления значения

4. **Template формы**:
   - Если template динамически рендерит `import_types` из context - изменений не требуется
   - Если template использует hardcoded список - обновить его

5. **Redis lock**:
   - Существующий механизм lock ([views.py:110-121](../../../backend/apps/integrations/views.py#L110-L121)) работает для всех типов
   - Изменений не требуется

import pytest
from django.test import TestCase
from apps.products.models import Product
from apps.integrations.views import _validate_dependencies

    """Тесты валидации зависимостей между типами импорта."""
