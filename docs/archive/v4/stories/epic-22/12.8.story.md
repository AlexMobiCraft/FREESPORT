## Story

**As a** Frontend разработчик,
**I want** реализовать страницу каталога товаров согласно новому макету Design System v2.0,
**so that** пользователи FREESPORT могли эффективно просматривать, фильтровать и сортировать товары с современным единообразным визуальным стилем, соответствующим монохромной графитовой цветовой схеме.

## Acceptance Criteria

1. Страница каталога создана с правильной структурой согласно Design System v2.0:
   - Layout: Sidebar фильтров слева (280px) + Grid товаров справа
   - Background: canvas (#F5F7FB)
   - Container: max-w-[1280px] mx-auto
   - Padding: desktop 24px, tablet 16px, mobile 12px
   - Gap между sidebar и grid: 24px

2. Компонент Breadcrumbs интегрирован:
   - Позиция: над заголовком страницы
   - Формат: Главная / СПОРТ / Единоборства / Одежда для единоборств
   - Использует компонент из Story 12.3
   - Последний элемент не кликабельный, выделен text-primary

3. Заголовок категории отображается:
   - Typography: headline-l (32px), font-weight: 700, text-primary
   - Margin bottom: 24px
   - Например: "Мужские худи и толстовки"

4. Описание категории (SEO-текст) отображается:
   - Typography: body-m (16px), text-secondary
   - Max 2-3 предложения
   - Line-clamp-3 с кнопкой "Показать больше"
   - Margin bottom: 32px

5. Sidebar фильтров интегрирован:
   - Использует компонент SidebarFilters из Story 12.5
   - Ширина: 280px
   - Sticky позиционирование: top-84px (Header 60px + отступ 24px)
   - Фильтры: Категории (древовидные), Бренды, Цена (range slider), Размер, Цвет
   - Кнопки "Применить" и "Сбросить" внизу
   - Адаптивность: на mobile скрывается, открывается модальным окном

6. Toolbar над Grid товаров реализован:
   - Левый блок: "Показано X из Y товаров" (body-m, text-secondary)
   - Центр: Переключатель grid/list view (иконки ⊞ и ≡)
   - Правый блок: SortSelect (Сортировка)
   - Height: 56px
   - Background: white
   - Border bottom: 1px solid #E3E8F2
   - Padding: 12px 24px
   - Адаптивность: на mobile упрощенный layout (только счетчик товаров и сортировка)

7. Grid товаров реализован:
   - Использует ProductCard из Story 12.4
   - Layout переключение: grid (default) и list views
   - Grid view: grid-cols-4 (desktop), grid-cols-3 (tablet), grid-cols-2 (mobile)
   - Gap: 16px между карточками
   - List view: 1 колонка, ProductCard в layout="list"
   - Загрузка с пагинацией (не infinite scroll)

8. Компонент Pagination интегрирован:
   - Позиция: под grid товаров
   - Использует компонент из Story 12.3
   - Формат: [←] [1] [2] [3] ... [18] [→]
   - Margin top: 48px
   - Center aligned

9. Active Filters Chips отображаются:
   - Позиция: над grid товаров, под toolbar
   - Использует компонент Chip из Story 12.5
   - Формат: "Бренд: Nike [X]", "Цена: 5000-10000₽ [X]"
   - Gap: 8px между chips
   - Кнопка "Очистить все фильтры" справа
   - Margin: 16px 0

10. SEO оптимизация реализована:
    - Next.js Metadata API с динамическим title и description
    - Title формат: "{Название категории} - FREESPORT"
    - Description: SEO-текст категории (первые 160 символов)
    - OpenGraph метатеги
    - Canonical URL
    - Structured data (JSON-LD) для товаров

11. Страница полностью адаптивна:
    - Desktop (>= 1024px): sidebar + grid 4 колонки
    - Tablet (641-1024px): sidebar + grid 3 колонки
    - Mobile (<= 640px): sidebar в модальном окне, grid 2 колонки
    - Padding: desktop 24px, tablet 16px, mobile 12px
    - Все интерактивные элементы доступны на touch устройствах

12. Все компоненты проходят accessibility проверку:
    - Semantic HTML (main, aside, section, nav)
    - Aria-labels для интерактивных элементов
    - Keyboard navigation (Tab, Enter, Escape)
    - Focus states видимы (ring-2 primary)
    - Skip to content link для screen readers
    - WCAG 2.1 AA compliance

13. Тесты созданы и проходят успешно:
    - Unit тесты для page component
    - Тесты фильтрации и сортировки
    - Тесты переключения grid/list view
    - Тесты pagination
    - Accessibility тесты
    - Покрытие >= 80%

### URL State Management [Source: Next.js best practices]

**Синхронизация фильтров с URL:**

```typescript
// Обновление URL при применении фильтров
const handleApplyFilters = (filters: FilterParams) => {
  const searchParams = new URLSearchParams();

  // Добавляем фильтры в query string
  if (filters.brandIds.length) {
    filters.brandIds.forEach((id) => searchParams.append("brand", id));
  }
  if (filters.priceMin) searchParams.set("priceMin", String(filters.priceMin));
  if (filters.priceMax) searchParams.set("priceMax", String(filters.priceMax));
  if (filters.sizes.length) {
    filters.sizes.forEach((size) => searchParams.append("size", size));
  }
  // ... остальные фильтры

  // Обновляем URL
  router.push(`/catalog/${categorySlug}?${searchParams.toString()}`);
};

// Парсинг фильтров из searchParams
const parseSearchParamsToFilters = (searchParams: any): FilterParams => {
  return {
    categoryIds: [],
    brandIds: Array.isArray(searchParams.brand)
      ? searchParams.brand
      : searchParams.brand
        ? [searchParams.brand]
        : [],
    priceMin: searchParams.priceMin ? Number(searchParams.priceMin) : undefined,
    priceMax: searchParams.priceMax ? Number(searchParams.priceMax) : undefined,
    sizes: Array.isArray(searchParams.size)
      ? searchParams.size
      : searchParams.size
        ? [searchParams.size]
        : [],
    colors: Array.isArray(searchParams.color)
      ? searchParams.color
      : searchParams.color
        ? [searchParams.color]
        : [],
  };
};
```

### Security Considerations

**XSS Protection:**

- React автоматически экранирует все данные из API через JSX
- **ВАЖНО**: Избегать использования `dangerouslySetInnerHTML` для пользовательского контента
- Все динамические данные (названия товаров, категории, описания) рендерятся через безопасные React компоненты

**URL Parameter Validation:**

- Валидировать все `searchParams` перед передачей в API
- Проверка типов и диапазонов для числовых параметров (price_min, price_max, page)
- Санитизация строковых параметров (category slug, brand IDs)
- Пример валидации:
  ```typescript
  const priceMin = searchParams.priceMin
    ? Math.max(0, Number(searchParams.priceMin))
    : undefined;
  const page = searchParams.page
    ? Math.max(1, Math.min(1000, Number(searchParams.page)))
    : 1;
  ```

**Rate Limiting:**

- Backend имеет rate limiting для `/api/v1/catalog/*` endpoints (обеспечивается Django Throttling)
- Защита от скрапинга каталога на уровне API

**CSRF Protection:**

- Не требуется для GET запросов каталога (read-only операции)
- POST запросы (добавление в корзину) защищены Django CSRF middleware

**Client-Side Data Exposure:**

- Публичные данные (товары, цены) - риск минимален
- Не передавать sensitive информацию в URL или localStorage
- B2B персональные цены доступны только авторизованным пользователям

### Storybook Integration (Optional Enhancement)

**Рекомендация:** Создать Storybook stories для всех новых компонентов каталога для улучшения документации и визуального тестирования.

**Компоненты для Storybook:**

- `CategoryHeader.stories.tsx` - заголовок категории с expandable описанием
- `CatalogToolbar.stories.tsx` - toolbar с grid/list toggle и сортировкой
- `ProductGrid.stories.tsx` - grid и list layouts с loading/empty states
- `ActiveFiltersChips.stories.tsx` - chips с различными фильтрами
- `MobileFiltersModal.stories.tsx` - модальное окно фильтров на mobile

**Преимущества Storybook:**

- Визуальная документация компонентов
- Изолированное тестирование UI
- Проверка различных states и props
- Демонстрация responsive behavior
- Упрощение code review

**Пример Storybook story:**

```typescript
// CategoryHeader.stories.tsx
import type { Meta, StoryObj } from "@storybook/react";
import { CategoryHeader } from "./CategoryHeader";

const meta: Meta<typeof CategoryHeader> = {
  title: "Catalog/CategoryHeader",
  component: CategoryHeader,
  parameters: {
    layout: "padded",
  },
};

export default meta;
type Story = StoryObj<typeof CategoryHeader>;

export const Default: Story = {
  args: {
    title: "Мужские худи и толстовки",
    description:
      "Широкий выбор мужских худи и толстовок для спорта и отдыха...",
  },
};

export const WithLongDescription: Story = {
  args: {
    title: "Спортивная одежда",
    description:
      "Очень длинное описание категории, которое требует expandable функционал...",
  },
};

export const WithoutDescription: Story = {
  args: {
    title: "Новинки",
  },
};
```

**Testing frameworks** [Source: CLAUDE.md]:

- Vitest 2.1.5 для unit тестов
- @testing-library/react 16.3.0 для компонентных тестов
- MSW 2.12.2 для API mocking

**Test standards:**

- Каждый компонент тестируется отдельно
- Тестировать рендеринг с разными props
- Тестировать интерактивность (клики, keyboard, filters)
- Тестировать адаптивность где возможно
- Тестировать accessibility атрибуты
- Integration тесты для полного flow: фильтры → сортировка → pagination

**Coverage requirements:**

- Минимум 80% покрытие для всех компонентов

**Test commands:**

```bash
# Запуск тестов
npm test

# Запуск тестов с покрытием
npm run test:coverage

# Запуск конкретного теста
npx vitest run src/app/catalog/__tests__/page.test.tsx
```

**Примеры тестов:**

```typescript
// page.test.tsx
import { render, screen } from '@testing-library/react'
import CatalogPage from '../page'

describe('CatalogPage', () => {
  it('renders catalog page with breadcrumbs', async () => {
    const page = await CatalogPage({
      params: { slug: ['sport', 'fitness'] },
      searchParams: {}
    })

    render(page)

    expect(screen.getByRole('navigation', { name: /хлебные крошки/i })).toBeInTheDocument()
    expect(screen.getByRole('heading', { level: 1 })).toBeInTheDocument()
  })

  it('renders products grid', async () => {
    const page = await CatalogPage({
      params: { slug: ['sport'] },
      searchParams: {}
    })

    render(page)

    const productCards = screen.getAllByRole('article')
    expect(productCards).toHaveLength(12) // или сколько товаров на странице
  })
})

// CatalogToolbar.test.tsx
import { render, screen, fireEvent } from '@testing-library/react'
import { CatalogToolbar } from '../CatalogToolbar'

describe('CatalogToolbar', () => {
  const mockOnViewModeChange = vi.fn()
  const mockOnSortChange = vi.fn()

  it('renders product count', () => {
    render(
      <CatalogToolbar
        totalCount={210}
        displayedCount={12}
        viewMode="grid"
        onViewModeChange={mockOnViewModeChange}
        sortBy="popularity"
        onSortChange={mockOnSortChange}
      />
    )

    expect(screen.getByText(/показано 12 из 210 товаров/i)).toBeInTheDocument()
  })

  it('switches view mode on button click', () => {
    render(
      <CatalogToolbar
        totalCount={210}
        displayedCount={12}
        viewMode="grid"
        onViewModeChange={mockOnViewModeChange}
        sortBy="popularity"
        onSortChange={mockOnSortChange}
      />
    )

    const listButton = screen.getByRole('button', { name: /список/i })
    fireEvent.click(listButton)

    expect(mockOnViewModeChange).toHaveBeenCalledWith('list')
  })
})

// ProductGrid.test.tsx
import { render, screen } from '@testing-library/react'
import { ProductGrid } from '../ProductGrid'

describe('ProductGrid', () => {
  const mockProducts = [
    { id: 1, name: 'Product 1', price: 1000, image: '/img1.jpg' },
    { id: 2, name: 'Product 2', price: 2000, image: '/img2.jpg' }
  ]

  it('renders products in grid layout', () => {
    render(
      <ProductGrid
        products={mockProducts}
        viewMode="grid"
        onAddToCart={vi.fn()}
        onToggleFavorite={vi.fn()}
      />
    )

    const grid = screen.getByRole('list')
    expect(grid).toHaveClass('grid', 'grid-cols-4')

    const productCards = screen.getAllByRole('article')
    expect(productCards).toHaveLength(2)
  })

  it('renders products in list layout', () => {
    render(
      <ProductGrid
        products={mockProducts}
        viewMode="list"
        onAddToCart={vi.fn()}
        onToggleFavorite={vi.fn()}
      />
    )

    const list = screen.getByRole('list')
    expect(list).toHaveClass('flex', 'flex-col')
  })

  it('renders loading state', () => {
    render(
      <ProductGrid
        products={[]}
        viewMode="grid"
        loading={true}
        onAddToCart={vi.fn()}
        onToggleFavorite={vi.fn()}
      />
    )

    const skeletons = screen.getAllByTestId('product-skeleton')
    expect(skeletons).toHaveLength(12) // количество skeleton placeholders
  })

  it('renders empty state', () => {
    render(
      <ProductGrid
        products={[]}
        viewMode="grid"
        loading={false}
        onAddToCart={vi.fn()}
        onToggleFavorite={vi.fn()}
      />
    )

    expect(screen.getByText(/товары не найдены/i)).toBeInTheDocument()
  })
})
```
