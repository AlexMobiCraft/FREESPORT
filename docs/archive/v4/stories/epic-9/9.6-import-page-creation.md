# Story 9.6: Создание новой страницы "Импорт из 1С"

**Epic:** Epic 9 - Advanced Admin Panel  
**Story ID:** 9.6  
**Status:** Draft  
**Priority:** Высокий (исправление недочетов Story 9.5)

---

## Story

**As a** Platform Administrator,  
**I want** отдельную страницу для запуска импорта данных из 1С с выбором типа импорта,  
**so that** я могу запускать новый импорт без необходимости выбирать существующие сессии импорта.

---

## Business Context

**Проблема:**  
Текущая реализация (Story 9.5) требует выбора существующей сессии импорта перед запуском нового импорта, что нелогично и создает путаницу. Admin action "🚀 Запустить импорт из 1С" должен создавать новую сессию, а не изменять существующую.

**Решение:**  
Создать отдельную страницу "Импорт из 1С" в разделе ИНТЕГРАЦИИ, которая позволяет выбрать тип импорта и запустить новый процесс импорта, автоматически создавая новую ImportSession.

**Ценность:**

- Логичный UX - запуск импорта отделен от просмотра истории
- Уменьшение ошибок пользователей
- Соответствие паттернам Django Admin

---

## Acceptance Criteria

1. **Страница доступна по URL `/admin/integrations/import_1c/`**
   - Страница отображается в меню ИНТЕГРАЦИИ перед "Сессии импорта"
   - Доступна только авторизованным администраторам

2. **Форма отображает 4 типа импорта с описанием**
   - "Полный каталог" - товары, категории, цены, остатки, свойства
   - "Только остатки" - обновление остатков товаров
   - "Только цены" - обновление цен товаров
   - "Клиенты" - импорт контрагентов из 1С

3. **При выборе типа вызывается соответствующая management команда через Celery**
   - Полный каталог → `import_catalog_from_1c --file-type=all`
   - Только остатки → `import_catalog_from_1c --file-type=rests`
   - Только цены → `import_catalog_from_1c --file-type=prices`
   - Клиенты → `import_customers_from_1c`

4. **Создается ImportSession с правильным типом**
   - CATALOG для полного каталога
   - STOCKS для остатков
   - PRICES для цен
   - CUSTOMERS для клиентов

5. **Валидация зависимостей работает**
   - Импорт остатков/цен требует наличия товаров в БД
   - Отображается понятное сообщение об ошибке

6. **Redis lock предотвращает параллельные импорты**
   - Если импорт уже запущен, отображается предупреждение
   - Lock автоматически освобождается после завершения

7. **Пользователь видит сообщение с Task ID и Session ID после запуска**
   - Сообщение содержит ссылку на страницу сессий для отслеживания прогресса
   - Автоматический редирект на страницу сессий через 2 секунды (опционально)

8. **Отображение прогресса импорта**
   - После запуска пользователь перенаправляется на страницу сессий
   - Страница сессий показывает статус Celery задачи в реальном времени
   - JavaScript автообновление обновляет статус каждые 5 секунд
   - Визуальная индикация: ⏳ В очереди → ▶️ Выполняется → ✅ Завершено / ❌ Ошибка
   - Пользователь видит что процесс активен и не завис

```python
# backend/apps/products/models.py

class ImportSession(models.Model):
    class ImportType(models.TextChoices):
        CATALOG = "catalog", "Каталог товаров"
        STOCKS = "stocks", "Остатки"
        PRICES = "prices", "Цены"
        CUSTOMERS = "customers", "Клиенты"

    class ImportStatus(models.TextChoices):
        STARTED = "started", "Начат"
        COMPLETED = "completed", "Завершен"
        FAILED = "failed", "Ошибка"

    import_type = models.CharField(max_length=20, choices=ImportType.choices)
    status = models.CharField(max_length=20, choices=ImportStatus.choices)
    celery_task_id = models.CharField(max_length=100, blank=True)
    started_at = models.DateTimeField(auto_now_add=True)
    finished_at = models.DateTimeField(null=True, blank=True)
    report_details = models.JSONField(default=dict, blank=True)
    error_message = models.TextField(blank=True)
```
