## 1. Введение

### Назначение системы импорта

Система импорта предназначена для синхронизации данных между 1С:Предприятие и платформой FREESPORT.

### Что импортируется

- Товары и категории
- Цены и остатки
- Контрагенты (клиенты)
- Свойства товаров

### Требования

- Права администратора Django Admin
- Доступ к разделу ИНТЕГРАЦИИ
- Подготовленные файлы выгрузки из 1С в формате CommerceML 3.1

## 2. Типы импорта

### 2.1 Полный каталог

**Когда использовать:** Первичная загрузка или полная синхронизация

**Что импортируется:**

- Товары (goods\_\*.xml)
- Торговые предложения (offers\_\*.xml)
- Цены (prices\_\*.xml)
- Остатки (rests\_\*.xml)
- Категории (groups.xml)
- Справочники (units.xml, storages.xml, priceLists.xml)
- Свойства товаров (propertiesGoods/, propertiesOffers/)

**Время выполнения:** ~15-30 минут для ~6200 товаров

**Зависимости:** Нет

**Файлы:**

```
data/import_1c/
├── goods/goods_1_*.xml
├── offers/offers_*.xml
├── prices/prices_1_*.xml
├── rests/rests_1_*.xml
├── groups/groups_1_1_*.xml
├── units.xml
├── storages.xml
├── priceLists.xml
├── propertiesGoods/
└── propertiesOffers/
```

---

### 2.2 Только остатки

**Когда использовать:** Ежедневное обновление остатков

**Что импортируется:**

- Остатки товаров на складах (rests\_\*.xml)

**Время выполнения:** ~2-5 минут

**⚠️ Зависимости:** Требуется предварительный импорт каталога

**Файлы:**

```
data/import_1c/rests/rests_1_*.xml
```

---

### 2.3 Только цены

**Когда использовать:** Обновление цен без изменения остатков

**Что импортируется:**

- Цены товаров (prices\_\*.xml)

**Время выполнения:** ~2-5 минут

**⚠️ Зависимости:** Требуется предварительный импорт каталога

**Файлы:**

```
data/import_1c/prices/prices_1_*.xml
```

---

### 2.4 Клиенты

**Когда использовать:** Синхронизация базы клиентов

**Что импортируется:**

- Контрагенты из 1С (contragents.xml)

**Время выполнения:** ~1-3 минуты

**Зависимости:** Нет

**Файлы:**

```
data/import_1c/contragents/contragents.xml
```

## 3. Как запустить импорт

### Шаг 1: Подготовка файлов

1. Выполните выгрузку из 1С в формате CommerceML 3.1
2. Разместите файлы в директории: `/path/to/data/import_1c/`
3. Убедитесь что файлы доступны для чтения

### Шаг 2: Запуск импорта

1. Откройте Django Admin: `http://your-domain.com/admin/`
2. Авторизуйтесь с правами администратора
3. Перейдите в раздел **ИНТЕГРАЦИИ** → **Импорт из 1С**
4. Выберите нужный тип импорта (radio button)
5. Нажмите кнопку **"Запустить импорт"**

### Шаг 3: Подтверждение запуска

- Вы увидите сообщение: "Импорт запущен (Task ID: xxx, Session ID: yyy)"
- Автоматически перенаправитесь на страницу сессий через 2 секунды

## 4. Отслеживание прогресса

### Страница сессий

URL: `/admin/integrations/session/`

### Статусы Celery задачи

- **⏳ PENDING** - Задача в очереди, ожидает выполнения
- **▶️ STARTED** - Импорт выполняется
- **✅ SUCCESS** - Импорт завершен успешно
- **❌ FAILURE** - Произошла ошибка
- **🔄 RETRY** - Повторная попытка после ошибки

### Автообновление

- Страница автоматически обновляется каждые 5 секунд
- Обновляются только активные задачи
- После завершения автообновление останавливается

### Детали сессии

Кликните на ID сессии для просмотра:

- Тип импорта
- Время начала и окончания
- Длительность
- Детальный отчет (report_details)
- Сообщения об ошибках (если есть)

## 5. Интерпретация результатов

### Успешный импорт

**Статус:** ✅ SUCCESS

**Что проверить:**

- Количество импортированных записей в report_details
- Время выполнения
- Отсутствие ошибок

**Пример report_details:**

```json
{
  "products_created": 6200,
  "products_updated": 0,
  "categories_created": 150,
  "prices_updated": 6200,
  "stocks_updated": 6200,
  "execution_time": "15m 23s"
}
```

---

### Импорт с ошибками

**Статус:** ❌ FAILURE

**Что делать:**

1. Проверьте поле "Error Message"
2. См. раздел [Troubleshooting](#troubleshooting)
3. Исправьте проблему
4. Запустите импорт заново

## 6. Troubleshooting

### Ошибка: "Сначала импортируйте каталог"

**Причина:** Попытка импорта остатков/цен без каталога

**Решение:**

1. Запустите "Полный каталог"
2. Дождитесь завершения (статус SUCCESS)
3. Запустите импорт остатков/цен

---

### Ошибка: "Импорт уже запущен"

**Причина:** Параллельный импорт запрещен (Redis lock)

**Решение:**

1. Дождитесь завершения текущего импорта
2. Проверьте страницу сессий
3. Если импорт завис (>1 час), обратитесь к администратору

---

### Ошибка: "FileNotFoundError: rests.xml"

**Причина:** Отсутствует файл выгрузки

**Решение:**

1. Проверьте наличие файлов в `/path/to/data/import_1c/`
2. Выполните выгрузку из 1С заново
3. Убедитесь в правах доступа к файлам (chmod 644)

---

### Импорт завис (статус STARTED >1 час)

**Причина:** Celery worker упал или зависла задача

**Решение:**

1. Проверьте логи Celery worker:
   ```bash
   docker-compose logs celery-worker
   ```
2. Перезапустите Celery worker:
   ```bash
   docker-compose restart celery-worker
   ```
3. Redis lock автоматически освободится через 1 час
4. Запустите импорт заново

---

### Ошибка: "IntegrityError: duplicate key"

**Причина:** Попытка создать товар с существующим external_id

**Решение:**

1. Проверьте данные в 1С на дубликаты
2. Используйте параметр `--clear-existing` для полной перезагрузки
3. Обратитесь к администратору для ручной очистки

## 7. FAQ

### Как часто запускать импорт?

- **Полный каталог:** 1 раз в неделю или при изменении структуры
- **Остатки:** Ежедневно или несколько раз в день
- **Цены:** При изменении прайс-листа
- **Клиенты:** 1 раз в день или при необходимости

### Можно ли запустить несколько импортов одновременно?

Нет, система блокирует параллельные импорты через Redis lock для предотвращения конфликтов данных.

### Что делать если нужно срочно запустить импорт?

Если текущий импорт можно прервать:

1. Обратитесь к администратору
2. Администратор может удалить сессию через Django Admin
3. Redis lock освободится
4. Запустите новый импорт

### Где посмотреть детальные логи импорта?

- **Django Admin:** детали сессии (report_details, error_message)
- **Celery logs:** `docker-compose logs celery-worker`
- **Django logs:** `docker-compose logs backend`

### Что делать если импорт работает слишком долго?

1. Проверьте размер файлов выгрузки
2. Проверьте нагрузку на сервер (CPU, RAM)
3. Рассмотрите увеличение `--chunk-size` в management команде
4. Обратитесь к администратору для оптимизации

### Можно ли откатить импорт?

Нет автоматического отката. Рекомендации:

1. Делайте backup БД перед полным импортом
2. Используйте `--dry-run` для тестирования
3. Для отката обратитесь к администратору
