# Story 9.5: Выборочный импорт данных из 1С через админ-панель

**Epic:** Epic 9 - Advanced Admin Panel  
**Story ID:** 9.5  
**Status:** Ready for Implementation  
**Priority:** Текущий спринт

---

## Story

**As a** Platform Administrator,  
**I want** гибкий интерфейс для выборочного импорта данных из 1С (каталог, остатки, цены, клиенты) с проверкой зависимостей,  
**so that** я могу обновлять только необходимые данные без полного импорта каталога.

---

## Business Context

**Проблема:**  
Текущий action "🚀 Запустить импорт каталога из 1С" всегда запускает полный импорт каталога (goods + offers + prices + rests), что занимает 2-5 минут. Администратору часто нужно обновить только остатки или цены без полного импорта.

**Решение:**  
Добавить UI с чекбоксами для выбора типов данных (каталог, остатки, цены, клиенты) с валидацией зависимостей.

**Ценность:**

- Сокращение времени обновления данных (остатки: 30 сек вместо 5 мин)
- Гибкость управления данными
- Снижение нагрузки на БД при частых обновлениях остатков

---

## Acceptance Criteria

### AC1: Структура админ-панели

- [ ] Старый action "trigger_catalog_import" заменен на "trigger_selective_import"
- [ ] Action переименован: "🚀 Запустить импорт каталога из 1С" → "🚀 Запустить импорт из 1С"
- [ ] При клике на action показывается intermediate page с формой выбора
- [ ] Подраздел "Сессии импорта" остается для мониторинга (readonly)

### AC2: UI для выбора типов данных

- [ ] Intermediate page содержит форму с чекбоксами:
  - ☑️ Каталог товаров (по умолчанию выбран)
  - ☐ Остатки товаров
  - ☐ Цены товаров
  - ☐ Клиенты
- [ ] Под каждым чекбоксом показаны подсказки:
  - "Остатки товаров" → "(требует наличия каталога)"
  - "Цены товаров" → "(требует наличия каталога)"
- [ ] Кнопки: "Запустить импорт" и "Отмена"

### AC3: Валидация зависимостей

- [ ] Если выбраны "Остатки" или "Цены" БЕЗ "Каталог":
  - Проверяется `Product.objects.exists()`
  - Если товаров нет → ошибка: "⚠️ Невозможно загрузить остатки/цены: каталог товаров пуст. Сначала импортируйте каталог."
- [ ] Если выбран "Каталог" → остатки и цены импортируются без проверки
- [ ] "Клиенты" не зависят от каталога

### AC4: Механика запуска импорта

- [ ] При выборе нескольких типов импорт выполняется последовательно:
  1. Каталог товаров (если выбран)
  2. Остатки товаров (если выбран)
  3. Цены товаров (если выбран)
  4. Клиенты (если выбран)
- [ ] Для каждого типа создается отдельная `ImportSession` с соответствующим `import_type`
- [ ] Distributed lock (Redis) предотвращает concurrent запуски
- [ ] После завершения показывается сводка:
  - "✅ Импорт завершен: Каталог (1234 товаров), Остатки (1234 SKU), Клиенты (56 клиентов)"

### AC5: Обработка ошибок

- [ ] Если импорт одного типа завершился с ошибкой → следующий тип НЕ запускается
- [ ] Показывается детальное сообщение: "❌ Ошибка импорта каталога: [детали]. Остатки и цены не импортированы."
- [ ] Все созданные `ImportSession` имеют корректный статус (`FAILED` для ошибочных)

## Technical Design

### Architecture Overview

**Подход:** Custom Admin Action с Intermediate Page (Django Admin нативный подход)

**Преимущества:**

- ✅ Не требует изменения моделей
- ✅ Полный контроль над UI
- ✅ Использует существующий Redis lock механизм

### Implementation Details

**Файлы для изменения:**

- `backend/apps/integrations/admin.py` - обновить action
- `backend/templates/admin/integrations/import_selection.html` - создать template

**Management Commands Mapping:**

| Тип импорта     | ImportType  | Management Command         | Параметры                          |
| --------------- | ----------- | -------------------------- | ---------------------------------- |
| Каталог товаров | `CATALOG`   | `import_catalog_from_1c`   | `--data-dir`, `--file-type=all`    |
| Остатки товаров | `STOCKS`    | `load_product_stocks`      | `--file` (rests.xml)               |
| Цены товаров    | `PRICES`    | `import_catalog_from_1c`   | `--data-dir`, `--file-type=prices` |
| Клиенты         | `CUSTOMERS` | `import_customers_from_1c` | `--file` (contragents.xml)         |

**Примечание:** Цены импортируются через `import_catalog_from_1c` с параметром `--file-type=prices`.

### Code Structure

```python
# backend/apps/integrations/admin.py

@admin.register(IntegrationImportSession)
class ImportSessionAdmin(admin.ModelAdmin):
    # ... existing code ...

    actions = ["trigger_selective_import"]

    @admin.action(description="🚀 Запустить импорт из 1С")
    def trigger_selective_import(self, request, queryset):
        """
        Запуск выборочного импорта с intermediate page для выбора типов.
        """
        if 'apply' in request.POST:
            # Обработка формы с чекбоксами
            selected_types = request.POST.getlist('import_types')

            # Валидация зависимостей
            is_valid, error_message = self._validate_dependencies(selected_types)
            if not is_valid:
                self.message_user(request, error_message, level='ERROR')
                return

            # Запуск импорта
            self._run_sequential_import(request, selected_types)
            return

        # Показываем intermediate page с чекбоксами
        context = {
            'title': 'Выбор типов данных для импорта',
            'import_types': [
                {'value': 'catalog', 'label': 'Каталог товаров', 'checked': True},
                {'value': 'stocks', 'label': 'Остатки товаров', 'hint': '(требует наличия каталога)'},
                {'value': 'prices', 'label': 'Цены товаров', 'hint': '(требует наличия каталога)'},
                {'value': 'customers', 'label': 'Клиенты'},
            ],
            'queryset': queryset,
            'action': 'trigger_selective_import',
        }
        return TemplateResponse(request, 'admin/integrations/import_selection.html', context)

    def _validate_dependencies(self, selected_types):
        """Проверка зависимостей между типами импорта"""
        if ('stocks' in selected_types or 'prices' in selected_types):
            if 'catalog' not in selected_types:
                # Проверяем наличие товаров в БД
                if not Product.objects.exists():
                    return False, (
                        "⚠️ Невозможно загрузить остатки/цены: каталог товаров пуст. "
                        "Сначала импортируйте каталог или выберите 'Каталог товаров' для импорта."
                    )
        return True, ""

    def _run_sequential_import(self, request, selected_types):
        """Последовательный запуск импортов"""
        redis_conn = get_redis_connection("default")
        lock_key = "import_catalog_lock"
        lock = redis_conn.lock(lock_key, timeout=3600)

        if not lock.acquire(blocking=False):
            self.message_user(
                request,
                "⚠️ Импорт уже запущен! Дождитесь завершения текущего импорта.",
                level='WARNING'
            )
            return

        results = []
        try:
            data_dir = getattr(settings, 'ONEC_DATA_DIR', None)
            if not data_dir:
                raise ValueError("Настройка ONEC_DATA_DIR не найдена в settings.")

            # Порядок импорта: catalog → stocks → prices → customers
            import_order = ['catalog', 'stocks', 'prices', 'customers']

            for import_type in import_order:
                if import_type not in selected_types:
                    continue

                try:
                    result = self._execute_import(import_type, data_dir)
                    results.append(result)
                except Exception as e:
                    # При ошибке прерываем цепочку
                    self.message_user(
                        request,
                        f"❌ Ошибка импорта {import_type}: {e}. "
                        f"Последующие импорты отменены.",
                        level='ERROR'
                    )
                    return

            # Формируем сводное сообщение
            summary = self._format_import_summary(results)
            self.message_user(request, f"✅ {summary}", level='SUCCESS')

        except Exception as e:
            self.message_user(request, f"❌ Критическая ошибка: {e}", level='ERROR')
        finally:
            lock.release()

    def _execute_import(self, import_type, data_dir):
        """Выполнение импорта конкретного типа"""
        if import_type == 'catalog':
            call_command('import_catalog_from_1c', '--data-dir', str(data_dir), '--file-type', 'all')
            return {'type': 'catalog', 'message': 'Каталог импортирован'}

        elif import_type == 'stocks':
            file_path = Path(data_dir) / 'rests' / 'rests.xml'
            call_command('load_product_stocks', '--file', str(file_path))
            return {'type': 'stocks', 'message': 'Остатки обновлены'}

        elif import_type == 'prices':
            call_command('import_catalog_from_1c', '--data-dir', str(data_dir), '--file-type', 'prices')
            return {'type': 'prices', 'message': 'Цены обновлены'}

        elif import_type == 'customers':
            file_path = Path(data_dir) / 'contragents' / 'contragents.xml'
            call_command('import_customers_from_1c', '--file', str(file_path))
            return {'type': 'customers', 'message': 'Клиенты импортированы'}

    def _format_import_summary(self, results):
        """Форматирование сводки результатов"""
        messages = [r['message'] for r in results]
        return f"Импорт завершен: {', '.join(messages)}"
```

```python
# backend/apps/integrations/admin.py

def _execute_import(self, import_type, data_dir):
    """Выполнение импорта конкретного типа с валидацией файлов"""
    if import_type == 'catalog':
        # Для каталога проверяем директорию
        if not Path(data_dir).exists():
            raise FileNotFoundError(f"Директория данных не найдена: {data_dir}")
        call_command('import_catalog_from_1c', '--data-dir', str(data_dir), '--file-type', 'all')
        return {'type': 'catalog', 'message': 'Каталог импортирован'}

    elif import_type == 'stocks':
        file_path = Path(data_dir) / 'rests' / 'rests.xml'
        if not file_path.exists():
            raise FileNotFoundError(
                f"Файл остатков не найден: {file_path}. "
                f"Убедитесь, что данные из 1С выгружены в {data_dir}"
            )
        call_command('load_product_stocks', '--file', str(file_path))
        return {'type': 'stocks', 'message': 'Остатки обновлены'}

    elif import_type == 'prices':
        if not Path(data_dir).exists():
            raise FileNotFoundError(f"Директория данных не найдена: {data_dir}")
        call_command('import_catalog_from_1c', '--data-dir', str(data_dir), '--file-type', 'prices')
        return {'type': 'prices', 'message': 'Цены обновлены'}

    elif import_type == 'customers':
        file_path = Path(data_dir) / 'contragents' / 'contragents.xml'
        if not file_path.exists():
            raise FileNotFoundError(
                f"Файл клиентов не найден: {file_path}. "
                f"Убедитесь, что данные из 1С выгружены в {data_dir}"
            )
        call_command('import_customers_from_1c', '--file', str(file_path))
        return {'type': 'customers', 'message': 'Клиенты импортированы'}
```
