# Story 16.2: История заказов с функцией повторного заказа

## Status

Done

## Story

**As a** зарегистрированный пользователь FREESPORT (B2B или B2C),
**I want** просматривать историю своих заказов с возможностью повторить предыдущий заказ,
**so that** я могу отслеживать свои покупки и быстро переза​казывать нужные товары

## Acceptance Criteria

1. Список заказов отображается с пагинацией (20 заказов на страницу)
2. Клик по заказу открывает детальную информацию
3. Фильтры работают без перезагрузки страницы (по статусу: все, новый, в обработке, отправлен, доставлен, отменён)
4. Кнопка "Повторить заказ" добавляет все товары в корзину и показывает уведомление
5. Для заказов с недоступными товарами показывается предупреждение
6. Экспорт заказа генерирует PDF документ (только для B2B пользователей)
7. E2E тест проверяет флоу повторного заказа

### API Specifications

[Source: docs/architecture/03-api-specification.md]

**Orders Endpoints:**

`GET /api/v1/orders/`

- Аутентификация: JWT Bearer Token (обязательна)
- Query параметры:
  - `status`: фильтр по статусу (pending, processing, shipped, delivered, cancelled)
  - `page`: номер страницы (default: 1)
  - `page_size`: размер страницы (default: 20, max: 100)
- Response 200:
  ```typescript
  {
    count: number;
    next: string | null;
    previous: string | null;
    results: Order[];
  }
  ```
- Response 401: Unauthorized

`GET /api/v1/orders/{id}/`

- Аутентификация: JWT Bearer Token (обязательна)
- Path параметр: `id` - ID заказа
- Response 200: Order объект (см. Data Models ниже)
- Response 401: Unauthorized
- Response 403: Forbidden (не ваш заказ)
- Response 404: Not Found

**Дополнительные endpoints (если нужны):**

`GET /api/v1/orders/{id}/export-pdf/`

- Аутентификация: JWT Bearer Token
- Доступно только для B2B пользователей
- Response 200: PDF file (application/pdf)
- Response 403: Forbidden (не B2B пользователь)

### Data Models

**Order Model:** [Source: docs/architecture/02-data-models.md]

```typescript
interface Order {
  id: number;
  order_number: string; // Уникальный номер заказа
  user: number; // ID пользователя (FK)

  // Статус заказа
  status:
    | "draft"
    | "pending"
    | "processing"
    | "shipped"
    | "delivered"
    | "cancelled"
    | "returned";

  // Суммы заказа
  subtotal: string; // Decimal (сумма товаров без налогов и доставки)
  tax_amount: string; // Decimal (НДС)
  shipping_cost: string; // Decimal (стоимость доставки)
  discount_amount: string; // Decimal (скидка)
  total_amount: string; // Decimal (итоговая сумма)

  // Адреса и платёжная информация
  shipping_address: {
    // JSON поле
    full_name: string;
    phone: string;
    city: string;
    street: string;
    building: string;
    apartment?: string;
    postal_code: string;
  };
  billing_address?: object; // JSON поле (опционально)
  payment_method: string;
  payment_status: "pending" | "paid" | "failed" | "refunded";

  // Интеграция с 1С
  onec_id?: string; // ID заказа в 1С
  exported_to_1c: boolean; // Флаг экспорта в 1С
  export_to_1c_at?: string; // ISO datetime
  last_sync_from_1c?: string; // ISO datetime

  // Системные поля
  created_at: string; // ISO datetime
  updated_at: string; // ISO datetime

  // Relations (expanded в API responses)
  items?: OrderItem[]; // Товары в заказе
}
```

```typescript
describe('OrderDetail', () => {
  it('adds all available products to cart when repeat order is clicked', async () => {
    // ARRANGE - подготовка данных
    const mockOrder = {
      id: 1,
      order_number: 'ORD-2025-001',
      items: [
        { product: 1, product_name: 'Product 1', quantity: 2, unit_price: '100' },
        { product: 2, product_name: 'Product 2', quantity: 1, unit_price: '200' }
      ]
    };

    const mockCartStore = {
      addItems: vi.fn()
    };

    server.use(
      rest.get('/api/v1/orders/1/', (req, res, ctx) =>
        res(ctx.json(mockOrder))
      )
    );

    // ACT - выполнение действия
    render(<OrderDetail order={mockOrder} onRepeatOrder={mockCartStore.addItems} />);

    const repeatButton = screen.getByRole('button', { name: /повторить заказ/i });
    await userEvent.click(repeatButton);

    // ASSERT - проверка результата
    await waitFor(() => {
      expect(mockCartStore.addItems).toHaveBeenCalledWith([
        { productId: 1, quantity: 2 },
        { productId: 2, quantity: 1 }
      ]);
    });

    expect(screen.getByText(/добавлено в корзину/i)).toBeInTheDocument();
  });
});
```

```typescript
addItems: (items: { productId: number; quantity: number }[]) => {
  set((state) => {
    const newItems = [...state.items];

    items.forEach(({ productId, quantity }) => {
      const existingItem = newItems.find(
        (item) => item.productId === productId,
      );

      if (existingItem) {
        existingItem.quantity += quantity;
      } else {
        newItems.push({ productId, quantity, price: 0 }); // price будет загружен с сервера
      }
    });

    return { items: newItems };
  });
};
```

### Additional Implementation Details

**Функция "Повторить заказ" - детальная логика:**

1. Получить список товаров из заказа (`order.items`)
2. Для каждого товара:
   - Найти текущий Product по `item.product` ID
   - Проверить `product.is_available`
   - Если доступен → добавить в список для корзины
   - Если недоступен → добавить в список недоступных
3. Если есть доступные товары:
   - Вызвать `cartStore.addItems(availableItems)`
   - Показать Toast success: "Добавлено X товаров в корзину"
4. Если есть недоступные товары:
   - Показать Toast warning с деталями недоступных товаров
   - Пример: "Товары 'Product A', 'Product B' недоступны"
5. Если все товары недоступны:
   - Показать Toast error: "Все товары из заказа недоступны"
   - Кнопка "Повторить заказ" disabled

**Экспорт в PDF - упрощённая реализация (MVP):**

Для MVP достаточно клиентской генерации PDF с базовой информацией:

- Использовать библиотеку `jspdf` или `react-pdf`
- Формат: A4, вертикальная ориентация
- Содержимое:
  - Заголовок: "Заказ №{order_number}"
  - Дата: {created_at}
  - Компания: {user.company_name}, ИНН: {user.tax_id}
  - Таблица товаров (наименование, артикул, кол-во, цена, сумма)
  - Итого: {total_amount}

Если требуется серверная генерация PDF:

- Создать backend endpoint `GET /api/v1/orders/{id}/export-pdf/`
- Использовать библиотеку `reportlab` (Python) для генерации PDF
- Вернуть PDF файл с `Content-Type: application/pdf`
