## Story

**As a** посетитель платформы FREESPORT,
**I want** видеть динамически загружаемые блоки с хитами продаж, новинками и популярными категориями на главной странице,
**so that** я могу быстро находить актуальные товары и популярные разделы каталога без необходимости искать их вручную.

## Acceptance Criteria

1. **Блок "Хиты продаж":**
   - Загружает данные из `GET /api/v1/products?is_hit=true&ordering=-created_at&page_size=8`
   - Отображает 8 товаров в горизонтальной прокрутке
   - Использует компонент `Card` из UI Kit (создаётся в Task 1.5)
   - Показывает `Badge` с автоматическим определением типа:
     - Приоритет 1: `variant="sale"` если `product.is_sale=true` (с `discount_percent%`)
     - Приоритет 2: `variant="promo"` если `product.is_promo=true`
     - Приоритет 3: `variant="new"` если `product.is_new=true`
     - Приоритет 4: `variant="hit"` если `product.is_hit=true`
     - Приоритет 5: `variant="premium"` если `product.is_premium=true`
   - Один товар показывает только ОДИН бейдж (по наивысшему приоритету)
   - Цвета бейджей соответствуют новой палитре (Story 14.4):
     - `sale`: bg `#FFE1E8`, text `#E53935`
     - `promo`: bg `#FFF0F5`, text `#FF2E93`
     - `new`: bg `#E7F3FF`, text `#0060FF`
     - `hit`: bg `#E0F5E8`, text `#00AA5B`
     - `premium`: bg `#F6F0E4`, text `#6D4C1F`
     - `discount`: bg `#F0E7FF`, text `#7C3AED`

2. **Блок "Новинки":**
   - Загружает данные из `GET /api/v1/products?is_new=true&ordering=-created_at&page_size=8`
   - Отображает 8 новых товаров в горизонтальной прокрутке
   - Использует компонент `Card` из UI Kit (создаётся в Task 1.5)
   - Показывает `Badge` с автоматическим определением типа (аналогично AC 1)
   - Для блока "Новинки" приоритет отдаётся `variant="new"` если только `is_new=true`
   - Цвета бейджей соответствуют новой палитре (Story 14.4):
     - `sale`: bg `#FFE1E8`, text `#E53935`
     - `promo`: bg `#FFF0F5`, text `#FF2E93`
     - `new`: bg `#E7F3FF`, text `#0060FF`
     - `hit`: bg `#E0F5E8`, text `#00AA5B`
     - `premium`: bg `#F6F0E4`, text `#6D4C1F`
     - `discount`: bg `#F0E7FF`, text `#7C3AED`

3. **Блок "Популярные категории":**
   - Загружает данные из `GET /api/v1/categories?parent_id__isnull=true&limit=6`
   - Отображает до 6 корневых категорий в grid layout
   - Каждая категория кликабельна и ведет на `/catalog/[slug]`
   - Показывает иконку категории и количество товаров

4. **Компоненты UI Kit:**
   - Использованы `Card`, `Badge` из `src/components/ui/`
   - Применены токены дизайн-системы (spacing, shadows, radius)
   - Соблюдена горизонтальная прокрутка с `scroll-snap` для товарных блоков

5. **Loading и Error states:**
   - Skeleton loaders отображаются во время загрузки данных
   - Error state с возможностью retry при ошибке API
   - Graceful degradation: при ошибке блок не отображается, но не ломает страницу

6. **Unit-тесты с MSW моками:**
   - Тесты для каждого компонента блока (HitsSection, NewArrivalsSection, CategoriesSection)
   - MSW моки для всех API эндпоинтов
   - Тестирование loading/error states
   - Покрытие кода 80%+

7. **Edge Cases и Graceful Degradation:**
   - **Пустые данные:** Если API возвращает пустой массив (0 товаров/категорий), блок НЕ отображается (не показываем пустой раздел)
   - **Товары без изображений:** Использовать placeholder изображение из `public/images/product-placeholder.png`
   - **Длинные названия:** Product name обрезается до 2 строк с ellipsis (`line-clamp-2`)
   - **API timeout:** После 10 секунд показывать error state с возможностью retry
   - **Частичные данные:** Если вернулось < 8 товаров, отображаем все доступные (не добавляем пустые карточки)

```yaml
Endpoint: GET /api/v1/products

  - in_stock: boolean (optional)

  next: string | null
  previous: string | null

```

**ВАЖНО:**

- Для "Новинок" используется фильтр `is_new=true` (Story 11.0)

**Product Schema:** [Source: docs/api-spec.yaml#components/schemas/Product]

```typescript
interface Product {
  id: number;
  name: string;
  slug: string;
  description: string | null;
  sku: string;
  brand: string;
  category_id: number;
  images: ProductImage[];
  price: Price;
  stock_quantity: number;
  is_in_stock: boolean;
  created_at: string; // ISO date
  updated_at: string;
  specifications?: Record<string, any>; // JSONB

  // Story 11.0: Маркетинговые флаги для бейджей

}

interface ProductImage {
  id: number;
  image: string; // URL
  alt_text: string;
  is_primary: boolean;
}

interface Price {
  retail: number;
  wholesale: {
    level1: number;
    level2: number;
    level3: number;
  } | null;
  currency: string; // "RUB"
  current: number; // Цена для текущего пользователя (учитывает роль)
```

```typescript
}
```

### API Client Configuration

**Централизованный HTTP клиент:** [Source: docs/architecture/tech-stack.md#Axios]

Story 11.2 использует общий `apiClient` из Epic 10, настроенный для работы с Django REST API:

```typescript
// src/services/api-client.ts (уже существует из Epic 10)
import axios, { AxiosError } from "axios";

export const apiClient = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL || "http://localhost:8001/api/v1",
  timeout: 10000,
  headers: {
    "Content-Type": "application/json",
  },
  withCredentials: true, // Для JWT cookie
});

// Response interceptor для единообразной обработки ошибок
apiClient.interceptors.response.use(
  (response) => response,
  (error: AxiosError) => {
    // Логирование ошибок в development
    if (process.env.NODE_ENV === "development") {
      console.error("API Error:", error.response?.data || error.message);
    }

    // 401 Unauthorized - можно добавить redirect на login (опционально)
    if (error.response?.status === 401) {
      // TODO: handle auth redirect if needed
    }

    return Promise.reject(error);
  },
);
```

**Использование в services:**

- Все API сервисы импортируют: `import { apiClient } from '@/services/api-client'`
- Path alias `@/` настроен в `tsconfig.json` → `./src`
- Base URL автоматически подставляется (без `/api/v1` в endpoint'ах)

**Переменные окружения:**

```typescript
  ordering?: string;
  page_size?: number;
  page?: number;
  category_id?: number;
  in_stock?: boolean;

export const productsService = {
  /**
   * Получить хиты продаж
   */
  async getHits(params?: Partial<GetProductsParams>): Promise<Product[]> {
    const { data } = await apiClient.get<ProductList>("/products", {
      params: {
        is_hit: true, // Story 11.0: фильтр хитов продаж
        ordering: "-created_at", // Сортировка по дате (консистентность)
        page_size: 8,
        in_stock: true,
        ...params,
      },
    });
    return data.results;
  },

  /**
   * Получить новинки
   */
  async getNew(params?: Partial<GetProductsParams>): Promise<Product[]> {
    const { data } = await apiClient.get<ProductList>("/products", {
      params: {
        is_new: true, // Story 11.0: фильтр новинок
        ordering: "-created_at", // Сортировка по дате (новые первые)
        page_size: 8,
        in_stock: true,
        ...params,
      },
    });
    return data.results;
  },
};
```

### Badge Logic

**Компонент ProductBadge с приоритетной логикой:**

```typescript
// src/components/common/ProductBadge.tsx
interface ProductBadgeProps {
  product: Product;
  className?: string;
}

/**
 * ProductBadge - компонент для автоматического определения и отображения бейджа товара
 *
 * Приоритет определения бейджа (от высшего к низшему):
 * 1. sale (распродажа с процентом скидки)
 * 2. promo (акция)
 * 3. new (новинка)
 * 4. hit (хит продаж)
 * 5. premium (премиум товар)
 *
 * Зависимость: Требует полей из Story 11.0
 */
export function ProductBadge({ product, className }: ProductBadgeProps) {
  // Приоритет 1: Распродажа с процентом скидки
  if (product.is_sale && product.discount_percent) {
    return (
      <Badge variant="sale" className={className}>
        {product.discount_percent}% скидка
      </Badge>
    );
  }

  // Приоритет 2: Акция
  if (product.is_promo) {
    return <Badge variant="promo" className={className}>Акция</Badge>;
  }

  // Приоритет 3: Новинка
  if (product.is_new) {
    return <Badge variant="new" className={className}>Новинка</Badge>;
  }

  // Приоритет 4: Хит продаж
  if (product.is_hit) {
    return <Badge variant="hit" className={className}>Хит</Badge>;
  }

  // Приоритет 5: Премиум
  if (product.is_premium) {
    return <Badge variant="premium" className={className}>Премиум</Badge>;
  }

  // Нет активных бейджей
  return null;
}
```

### Technical Constraints

1. **Performance:**
   - Lazy loading изображений через `next/image`
   - Optimistic rendering: показывать skeleton сразу, загружать данные параллельно
   - Мемоизация компонентов через `React.memo` если необходимо

2. **Accessibility:**
   - Горизонтальная прокрутка доступна через keyboard (tab navigation)
   - aria-label для блоков ("Хиты продаж", "Новинки")
   - Skeleton loaders имеют `role="status"` и `aria-busy="true"`

3. **SEO Considerations:**
   - SSG рендерит блоки на сервере с начальными данными
   - ISR (revalidate: 3600) обновляет контент каждый час
   - Structured data (JSON-LD) для товаров (опционально)

4. **Security (XSS Protection):**
   - **Product descriptions** рендерятся как plain text через React default escaping
   - **НЕ используется** `dangerouslySetInnerHTML` для отображения контента из API
   - **Изображения:** Только из trusted sources через `next/image` (автоматическая валидация domains)
   - **Category icons:** Emoji из API рендерятся через Unicode (безопасно)
   - Если в будущем потребуется HTML в descriptions → использовать DOMPurify для sanitization
