## 1. UX Goals & Principles

### Target User Personas

**Оптовый покупатель (B2B):**

- Ищет условия сотрудничества, минимальные заказы, систему скидок
- Нужна быстрая связь с менеджером
- Важна доставка до ТК

**Представитель спортивной организации:**

- Школы, федерации, ДЮСШ, фитнес-клубы
- Нужны документы для тендеров
- Особый интерес к условиям работы

**Новый посетитель:**

- Изучает компанию перед первым заказом
- Хочет понять преимущества и ценности бренда

### Usability Goals

1. **Быстрый доступ к ключевой информации** — условия доставки видны за 1 клик
2. **Прозрачность B2B условий** — чёткие суммы, адреса, контакты
3. **Доверие с первого взгляда** — статистика, ценности, гарантии

### Design Principles

1. **Консистентность с основным сайтом** — используем токены из дизайн-системы
2. **Информация > декор** — фокус на содержании, минимум визуального шума
3. **Mobile-first для тизеров** — на мобильных информация компактна, но полна
4. **CTA везде** — каждый блок ведёт к действию (заявка, контакт, переход)

## 3. User Flows

### Flow 1: B2B Партнёр — Знакомство и заявка

```mermaid
flowchart TD
    A[Посетитель на главной] --> B{Видит тизер «Почему FREESPORT?»}
    B -->|Клик CTA| C[/partners]
    B -->|Скролл вниз| D[Продолжает просмотр]
    D --> E{Footer или Header}
    E -->|Клик «Партнёрам»| C
    
    C --> F[Hero: Условия сотрудничества]
    F --> G[Изучает типы клиентов]
    G --> H[Читает условия]
    H --> I[Видит блок «Персональный менеджер»]
    I --> J{Есть вопросы?}
    J -->|Да| K[Открывает FAQ]
    J -->|Нет| L[Скролл к CTA]
    K --> L
    L --> M[Клик «Зарегистрироваться»]
    M --> N[Переход на /register?from=partners]
    N --> O[✅ Заявка отправлена]
```

### Flow 2: Посетитель — Проверка условий доставки

```mermaid
flowchart TD
    A[Посетитель] --> B{Откуда пришёл?}
    B -->|Главная| C[Тизер «Доставка»]
    B -->|Корзина| D[Ссылка в Checkout]
    B -->|Footer| E[Прямой переход]
    
    C --> F[/delivery]
    D --> F
    E --> F
    
    F --> G[Hero: Доставка по России]
    G --> H{Какой способ интересует?}
    H -->|ТК| I[Карточка: Доставка до ТК]
    H -->|Самовывоз| J[Карточка: Самовывоз + адрес]
    
    I --> K[Понял условия]
    J --> K
    K --> L{Остались вопросы?}
    L -->|Да| M[Контакты логистики]
    L -->|Нет| N[Возврат к покупкам]
```

### Flow 3: Посетитель — Знакомство с компанией

```mermaid
flowchart TD
    A[Новый посетитель] --> B[/about]
    B --> C[Hero: О компании]
    C --> D[Читает «Кто мы»]
    D --> E[Изучает ценности]
    E --> F[Видит статистику]
    F --> G{Впечатлён?}
    G -->|Да| H[CTA: Стать партнёром или В каталог]
    G -->|Нужно больше| I[Переход к /partners]
    H --> J[✅ Доверие сформировано]
    I --> J
```

### Существующие компоненты (переиспользуем)

|-----------|---------------|
| `Button` | CTA кнопки |

| `SupportPanel` | Карточки доставки |
| `Badge` | «БЕСПЛАТНО», статусы |
| `Input`, `Select` | Форма регистрации (существующая) |
| `Breadcrumb` | Навигация |

### Новые компоненты

#### 1. FeatureCard

**Purpose:** Карточка преимущества для тизеров

```typescript
interface FeatureCardProps {
  icon: LucideIcon;
  title: string;
  description?: string;
  variant?: 'default' | 'horizontal' | 'compact';
}
```

**Tokens:**

- `background: var(--color-neutral-100)`
- `radius: 16px`
- `iconSize: 48px`
- `iconBackground: var(--color-primary-subtle)`

#### 2. StatCounter

**Purpose:** Анимированный счётчик статистики

```typescript
interface StatCounterProps {
  value: number;
  suffix?: string;
  label: string;
  duration?: number;
}
```

**Behavior:** Анимация при scroll into view (Intersection Observer)

#### 3. ProcessSteps

**Purpose:** Шаги процесса «Как начать сотрудничество»

```typescript
interface ProcessStepsProps {
  steps: Array<{
    number: number;
    title: string;
    description: string;
  }>;
  variant?: 'numbered' | 'timeline';
}
```

#### 4. Accordion

**Purpose:** FAQ секция

```typescript
interface AccordionProps {
  items: Array<{
    question: string;
    answer: string;
  }>;
  allowMultiple?: boolean;
}
```

**ARIA:** `aria-expanded`, `aria-controls`

## 7. Accessibility

**Standard:** WCAG 2.1 Level AA

### Key Requirements

- Контраст текста ≥ 4.5:1
- Focus индикаторы на всех интерактивных элементах
- Keyboard navigation
- Heading hierarchy (один h1 на страницу)
- ARIA для Accordion (`aria-expanded`, `aria-controls`)
- `tel:` и `mailto:` ссылки для контактов

### Heading Structure

**`/about`:** h1 → h2 (Кто мы, Ценности, Статистика, CTA)  
**`/partners`:** h1 → h2 (Типы клиентов, Как начать, Менеджер, Рекламации, FAQ, CTA)  
**`/delivery`:** h1 → h2 (ТК, Самовывоз, Контакты)
